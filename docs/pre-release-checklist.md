# 本番前チェックリスト

## 1. 環境準備

- `.env` に `DATABASE_URL` が設定されている
- `npx prisma generate` が通る
- `npx prisma migrate dev` または `npx prisma db push` が通る
- `npm run db:seed` が必要なら投入済み
- `npm run lint` が通る
- `npx tsc --noEmit` が通る

## 2. 制度設定確認

- 評価制度設定
  - `自律成長力 / 協調相乗力` の項目が最新化されている
  - `根拠必須` と `重み` が意図通り設定されている
- 昇給ルール設定
  - `総合等級別昇給ルール` が設定されている
  - `期待充足ランク別昇給ルール` が補助基準として設定されている
- 給与構成設定
  - `自律成長基準額` が設定されている
  - `協調相乗基準額` が設定されている
  - `粗利補正` が設定されている

## 3. 評価フロー確認

- `/evaluations/my`
  - 自己評価を保存できる
  - `協調相乗力` で継続実践を選んだ場合、根拠入力が必要になる
- `/evaluations/team`
  - 上長評価を保存できる
  - 根拠が再表示される
- `/evaluations/finalize`
  - 最終評価を保存できる
  - `自律成長力達成率 / 協調相乗力実施率 / 総合等級 / 期待充足ランク` が表示される

## 4. 昇給シミュレーション確認

- `/salary/simulations`
  - `新月額(参考)` が給与構成と粗利補正から計算される
  - `決定額` を入力すると `昇給率 / 昇給額` が逆算される
  - 差額が大きい行で `調整理由` が必須になる
  - 保存できる
  - 承認できる
  - 反映できる

## 5. 結果確認

- `/salary/results`
  - 一覧で `参考額 / 決定額 / 差額 / 調整理由 / 状態` を確認できる
  - `状態` と `差額ありのみ` で絞り込める
  - CSV 出力できる
- `/salary/results/[userId]`
  - 個人詳細で評価と昇給決定をまとめて確認できる

## 6. 監査確認

- `/settings/audit-logs`
  - `SAVE_SALARY_SIMULATION`
  - `APPROVE_SALARY_SIMULATION`
  - `APPLY_SALARY_SIMULATION`
  が記録される
- 昇給シミュレーション関連ログで以下が確認できる
  - 対象人数
  - 差額あり件数
  - 理由未入力件数
  - 代表的な調整内容

## 7. 運用確認

- `期待充足ランク` を役割期待充足の補助指標として説明できる
- `B` は低評価ではなく、現在の役割期待を安定して満たしている状態として扱う認識が揃っている
- `総合等級` を主基準、`期待充足ランク` を補助基準として運用する認識が揃っている
- `調整理由` をどの水準で書くかの運用ルールがある

## 8. 推奨最終確認シナリオ

1. 評価制度設定を確認
2. 自己評価を入力
3. 上長評価を入力
4. 最終評価を確定
5. 昇給シミュレーションで決定額を入力
6. 差額が大きい行に調整理由を入力
7. 承認
8. 反映
9. 結果一覧と監査ログを確認
